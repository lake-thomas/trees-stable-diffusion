# Contributing to Trees SD

Thank you for your interest in contributing to Trees SD! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/trees-stable-diffusion.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install development dependencies: `pip install -r requirements.txt`
5. Install the package in development mode: `pip install -e .`

## Development Guidelines

### Code Style

- Follow PEP 8 style guidelines for Python code
- Use meaningful variable and function names
- Add docstrings to all public functions and classes
- Keep functions focused and single-purpose

### Testing

- Add tests for new features
- Ensure existing tests pass before submitting a PR
- Run tests with: `pytest tests/`

### Commit Messages

- Use clear and descriptive commit messages
- Start with a verb in present tense (e.g., "Add", "Fix", "Update")
- Keep the first line under 72 characters
- Add detailed description if needed

### Pull Requests

1. Update documentation if you're changing functionality
2. Add examples if you're adding new features
3. Ensure your code passes all tests
4. Update CHANGELOG.md with your changes
5. Submit a pull request with a clear description of your changes

## Areas for Contribution

### High Priority
- Additional dataset format support
- More comprehensive tests
- Performance optimizations
- Documentation improvements

### Feature Ideas
- Support for more Stable Diffusion versions
- Additional LoRA configurations
- Data augmentation options
- Inference/generation utilities
- Wandb/TensorBoard integration improvements

### Documentation
- Tutorial videos or blog posts
- Additional usage examples
- Translation of documentation

## Questions?

Feel free to open an issue for:
- Bug reports
- Feature requests
- Questions about usage
- Discussions about improvements

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and help them get started
- Focus on constructive feedback
- Respect different opinions and approaches

Thank you for contributing to Trees SD!
